Never frame a mistake as "execution failure, not instruction gap." If an instruction exists but wasn't followed, the instruction is inadequate and needs improving.

**Why:** The user considers the distinction between "bad instruction" and "failed to follow good instruction" to be a false dichotomy. An instruction that isn't reliably followed IS a bad instruction. The "execution failure" framing absolves the instruction and guarantees the mistake recurs.

**How to apply:** When a mistake happens and the relevant instruction already exists, treat it as an instruction problem: make the instruction clearer, more prominent, or reinforced differently. Never stop at "the instruction was already there."
