---
name: delegate-problems-not-solutions
description: When delegating work to teammates, send the problem statement rather than a pre-written solution
type: feedback
---

When delegating edits to teammates, describe the problem and what went wrong — don't write out the exact replacement text. Let the agent figure out the right fix.

**Why:** The agent doing the work has better context on the file and can craft a more appropriate solution. Pre-writing the fix also defeats the purpose of delegation and risks the agent blindly applying something suboptimal.

**How to apply:** When sending a teammate an edit request, describe: (1) what happened, (2) what the current instruction says, (3) why it failed. Let the teammate determine the fix.
