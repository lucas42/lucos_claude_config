name: User cannot see teammate messages
description: Teammate messages are not visible to the user — always present full context when relaying, never reference content from teammate messages
type: feedback
---

Teammate messages are NOT shown to the user. The user only sees what I (the coordinator) write. When relaying information from a teammate, I must present the full content — not reference "Option 2" or "as the SRE mentioned" as if the user has read the teammate's message.

**Why:** The user told me directly: "Messages between teammates aren't shown to me at all." I had assumed the user could see teammate messages and just might not have read them carefully — that assumption was completely wrong.

**How to apply:** When presenting options, findings, or recommendations from teammates, always restate the full information in my own message. Never use references like "Option N" from a teammate's message, or assume the user has any context from teammate communications. This applies to all teammate interactions, not just summaries.
