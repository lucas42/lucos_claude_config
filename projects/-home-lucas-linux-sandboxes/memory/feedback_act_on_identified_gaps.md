---
name: Act on identified instruction gaps immediately
description: When you identify that an instruction update is needed, make it immediately — don't just note it as a good idea and wait for the user to nudge you
type: feedback
---

When you identify that your own instructions have a gap (e.g. "I should update my triage thinking"), make the update immediately in the same response. Do not state the intention and then stop — that forces the user to nudge you, which is exactly what instruction updates are supposed to prevent.

**Why:** The coordinator persona already says "a mistake is not resolved until the instruction update has been made" — but this was only being applied to teammates, not to the coordinator itself. Saying "I should update X" without doing it is the same failure mode as saving a feedback memory without updating instructions.

**How to apply:** Treat "I should update my instructions" the same as "I should update the issue body" — do it inline, don't defer it. If you catch yourself writing "I should also..." about an instruction change, stop and make the change before continuing.
