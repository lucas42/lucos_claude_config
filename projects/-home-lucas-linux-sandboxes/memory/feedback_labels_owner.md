---
name: Labels managed by issue-manager
description: Label creation and management should be routed to lucos-issue-manager, not other personas
type: feedback
---

When labels need to be created or managed on any repo, route to `lucos-issue-manager` — not `lucos-system-administrator` or other personas.

**Why:** The issue manager owns the label taxonomy and conventions. Other personas may create labels ad-hoc without following the standard set.

**How to apply:** Any time a missing label is discovered or labels need to be added to a repo, delegate to `lucos-issue-manager`.
