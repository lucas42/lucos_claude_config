---
name: Don't auto-revert agent work
description: When user says an agent shouldn't have done something, ask whether to revert rather than assuming revert is needed
type: feedback
---

When the user states or suggests that an agent shouldn't have done something (e.g. picked up unprioritised work, made a change without instructions), do NOT interpret that as an instruction to revert the change. Instead, check with the user whether:
1. The work needs to be reverted, or
2. We just need to update the process to prevent it happening again

Particularly when an agent has picked up non-prioritised work — yes, we want to stop them bypassing the process, but there's no need to throw out work they've already spent time on. The default should be to keep the work and fix the process, unless the user explicitly asks for a revert.
