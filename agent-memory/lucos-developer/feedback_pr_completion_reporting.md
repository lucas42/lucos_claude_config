---
name: Reporting PR completion: just say approved
description: After PR approval, report "PR approved" + URL only — do not determine or report supervised/unsupervised status
type: feedback
---

After a PR is approved, report back to team-lead with just the PR URL and that it has been approved. Do not add "awaiting lucas42", "auto-merging", or any supervised/unsupervised language — in the approval report OR in any other message (status updates, multi-PR chain descriptions, etc.).

**Why:** Multiple incidents of wrong reporting — lucos_photos flagged as supervised three times in one session when it is unsupervised. The coordinator always runs check-unsupervised itself — this is not the developer's job.

**How to apply:** Never use the words "supervised", "unsupervised", "needs lucas42", or "auto-merge" in any message to team-lead, regardless of context. Report: "PR {url} is approved." Stop there.
