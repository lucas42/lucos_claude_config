When editing Luke's CV (any file in `lucas42/lukeblaney_cv`):

**Proceed without asking** (mechanical edits):
- Spelling corrections
- Formatting tweaks (whitespace, list syntax, escaping)
- Standardising section headings to ATS-recognised names (e.g. "Talks & Panels" → "Publications & Talks")
- Build pipeline / config changes
- `.gitignore` updates

**Stop and consult Luke first** (copy decisions):
- Rewording any sentence or bullet
- Dropping or merging whole sections
- Adding new claims (employers, dates, achievements, skills)
- Changing the substance of a description
- Reordering experience

**Why:** Luke wrote the CV in his own voice. He's happy for me to fix mechanical issues but doesn't want me drifting his prose. Stated 2026-05-19.

**How to apply:** If a fix is purely surface-level (spelling, format, heading rename) — just do it. If it touches what's being *said* — pause, summarise the change, and ask.
